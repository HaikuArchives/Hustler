What is it
Hustler is a simple audio player for BeOS R4.5 or higher that runs inside the Deskbar. It uses add-ons to do the actual playing, so what kinds of audio are supported depends on the add-ons that you have installed. The standard distribution comes with a CDDA player add-on and an experimental SoundPlay remote control add-on.

How to install it
Double-click the Installer icon to launch the installation program. The Installer will automatically put Hustler in the "/boot/home/config/bin/" folder and the add-ons in the "/boot/home/config/add-ons/Hustler/" folder. If you wish, the Installer will also modify your boot script to launch Hustler every time you start your computer.

How to get rid of it
Double-click the Uninstaller icon to launch the uninstallation program. The Uninstaller will automatically remove Hustler and its add-ons from your system and your boot script.

How to write add-ons
If you're a developer who wishes to create a add-on for Hustler (in which case you probably have way too much spare time ;-), you may want to read the Hustler add-on API documentation.

Known problems
Because of problems with the CD driver that ships with the BeOS, I have not been able to implement some features in the CDDA add-on, such as stopping a CD or skipping through tracks.

It may take a few seconds for the CDDA add-on to recognize a CD that contains a data track if you have enabled automatic disk mounting in the BeOS Disk Mount Settings menu.

The SoundPlay remote control add-on is very limited, and may cause SoundPlay to hang from time to time.

History
Version 1.0 (Saturday, 20 March 1999)

  • First version.
Version 2.0 (Saturday, 21 August 1999)
  • Hustler is no longer limited to just playing CDs; it now uses add-ons to deal with an unlimited variety of players.
  • Added the "volume" and "panning" menu items.
  • On start-up, Hustler automatically activates the player that you selected most recently.
Version 2.1 (Saturday, 25 September 1999)
  • Fixed an error in the makefiles of the add-ons that prevented them from compiling correctly under BeOS R4.5.2.
  • The Installer script now correctly cleans up when some error occurs.
  • Hustler doesn't hang anymore if something goes wrong with the SoundPlay remote control add-on.
Version 2.2 (Tuesday, 28 March 2000)
  • New release for BeOS R5. Older versions of Hustler came with source code only, which was automatically compiled during installation. However, because of header file changes between R5 and older versions of the BeOS, and because the free version of BeOS R5 doesn't necessarily include development tools, Hustler 2.2 is already compiled. This means that it only runs on the Intel platform, since I don't have a PowerPC machine.
